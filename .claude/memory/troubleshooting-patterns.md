# Troubleshooting Patterns — my-blog

> 블로그 작성 중 발생한 문제 패턴 기록
> 등록: `/blog-troubleshoot <문제 설명>`

---

## 디버깅 프로토콜

문제 발생 시:
1. 기존 패턴 확인 (이 파일 검색)
2. 증상 → 원인 → 해결 순으로 분석
3. 해결 후 패턴 등록

---

## 패턴 목록

(아직 등록된 패턴 없음)

---

## 등록 형식

```markdown
### [TS-NNN] 제목

**증상**: 어떤 오류/문제가 발생했는가
**원인**: 왜 발생했는가
**해결**: 어떻게 해결했는가
**관련 주제**: docker-swarm / spring-seaweedfs / spring-temporal
**날짜**: YYYY-MM-DD
```
