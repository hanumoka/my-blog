# 블로그 작성 규칙

## 필수 구성 요소

모든 블로그 파일은 다음 구조를 따른다:

```markdown
# 제목 (명확하고 검색 친화적)

> **난이도**: 입문/중급/고급
> **소요 시간**: N분
> **사전 지식**: [필요한 배경 지식]

## 개요

## 핵심 개념

## 환경 설정 (해당 시)

## 실습

## 요약

## 참고 자료
```

## 파일 네이밍

- 형식: `{순번:01}-{주제-kebab-case}.md`
- 순번은 폴더 내 기존 파일 수 확인 후 다음 번호 부여
- 예: `01-overview.md`, `02-installation.md`

## 스타일 규칙

### 언어
- 본문: 한국어
- 코드 블록 내용: 원문 그대로
- 기술 용어: 원문 병기 가능 (`컨테이너(Container)`)

### 코드 예제
- 언어 명시 필수: ` ```yaml `, ` ```bash `, ` ```java `
- 복사해서 바로 실행 가능한 완성 형태
- 비밀번호/키: `YOUR_PASSWORD`, `YOUR_API_KEY`로 마스킹
- 버전 번호는 항상 명시

### 참고 자료 형식
```markdown
## 참고 자료

- [제목](URL) — 공식 문서
- [제목](URL) — 버전 X.X 기준
```

## 금지 사항

- 구버전 예제를 버전 명시 없이 사용
- 출처 불명 정보 단독 기재
- 실행 불가능한 불완전 코드 예제
- 보안 정보(실제 키/비밀번호) 포함
