블로그 학습 가이드 작성 워크플로우를 실행합니다.

## 워크플로우

### 1단계: 주제 구체화

$ARGUMENTS 분석:
- 주제가 명확하면 → 진행
- 모호하면 → STOP + 구체화 질문 (어떤 기능? 어느 수준? 사전 지식 가정?)

### 2단계: 웹 검색 (필수)

다음 순서로 최신 정보 수집:
1. **공식 문서** 검색 (최신 버전 확인)
2. **공식 GitHub** 릴리즈 노트 확인
3. **커뮤니티 자료** (날짜 확인 후 인용)

수집 항목:
- 현재 최신 버전
- 핵심 개념/아키텍처
- 실습에 필요한 설정/코드
- 주의사항/알려진 이슈

### 3단계: 구조 설계

학습 가이드 구조:
```
# 제목

> 난이도 / 소요 시간 / 사전 지식

## 개요
- 이 글에서 배우는 것
- 완성 후 결과물

## 핵심 개념
- 개념 설명 (다이어그램/비유 활용)

## 환경 설정
- 버전 명시
- 설치/설정 방법

## 실습
- 단계별 구현 (복사해서 바로 실행 가능한 완성 코드)

## 요약
- 핵심 내용 3~5줄

## 참고 자료
- 출처 URL 목록
```

### 4단계: 작성

규칙:
- 본문: 한국어
- 코드/명령어/기술 용어: 원문 그대로
- 예제 코드: 동작하는 완성 형태
- 비밀번호/키: `YOUR_PASSWORD`, `YOUR_API_KEY` 마스킹

### 5단계: 저장

파일 경로: `{폴더}/{순번:01}-{주제-kebab-case}.md`

예시:
- `docker-swarm/01-docker-swarm-overview.md`
- `spring-temporal/02-workflow-basics.md`

### 6단계: 상태 갱신

`blog-claude-docs/session/current-status.md`에 완료 항목 추가:
```
- [x] {제목} (`{파일경로}`) — YYYY-MM-DD
```

## 주의사항
- 웹 검색 없이 작성 금지 (최신 정보 필수)
- 구버전 예제 사용 금지 (버전 명시 항상)
- 출처 없는 정보 단독 기재 금지
